## 📋 截图技巧

### 推荐工具
- **Windows**: Snipping Tool, PrtSc
- **Mac**: Command+Shift+4
- **Linux**: Screenshots, gnome-screenshot

### 图片要求
1. **清晰度**: 确保文字可读
2. **完整性**: 包含完整的上下文
3. **标注**: 可以添加红框或箭头突出重点
4. **格式**: PNG或JPG格式
5. **大小**: 单张不超过5MB

### Word文档组织
```
1. 封面页 - 基本信息
2. 技术概述 - 配置截图
3. 特征构造过程 - 代码截图 + 运行截图
4. 效果验证 - 性能对比截图
5. 附录 - 详细代码和日志截图
```

## ⚠️ 注意事项

1. **隐私保护**: 
   - 遮蔽真实的API密钥
   - 可以用马赛克或黑块覆盖敏感信息

2. **完整性**: 
   - 每个主要特征至少要有一个生成过程截图
   - 确保prompt和输出的完整展示

3. **真实性**: 
   - 所有截图必须来自实际运行过程
   - 不要使用伪造或修改的截图

4. **时间戳**: 
   - 保留日志中的时间戳信息
   - 确保时间逻辑合理
